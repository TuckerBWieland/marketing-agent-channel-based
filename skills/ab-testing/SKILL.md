# A/B Testing

Cross-channel testing methodology.

## When to Test

- Before scaling any campaign or content format
- When performance plateaus and you need to find a new lever
- When the team disagrees — let data decide

## Testing Framework

### 1. Hypothesis
Write it down: "Changing [variable] from [A] to [B] will [increase/decrease] [metric] by [expected amount] because [reason]."

### 2. Design
- **One variable per test** — isolate the change
- **Control vs. variant** — always have a baseline
- **Sample size**: Use a significance calculator. Minimum [1,000] exposures per variant for most channels.
- **Duration**: Run for at least one full business cycle (typically 7 days)

### 3. Measure
- Primary metric: The ONE metric that determines the winner
- Secondary metrics: Watch for unintended effects (e.g., higher CTR but lower conversion)
- Statistical significance threshold: **90%+** before declaring a winner

### 4. Act
- Implement the winner
- Document the result (what, why, magnitude)
- Plan the next test

## What to Test by Channel

| Channel | High-impact variables |
|---------|---------------------|
| Email | Subject line, send time, CTA copy, preview text |
| Social | Hook/first line, image vs. video, posting time |
| Blog | Title, CTA placement, content length |
| Paid | Audience, creative, headline, CTA, landing page |

## Common Mistakes

- Ending tests too early (small samples = noise, not signal)
- Testing multiple variables at once (can't isolate what worked)
- Not documenting results (you'll re-run the same test later)
- Ignoring secondary metrics (winner on CTR, loser on conversions)
